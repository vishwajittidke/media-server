**Version:** 2.0

**Status:** Ready for Development

**Owner:** Vishwajit Tidke

---

# 1. Product Vision

Build a modern, secure, self-hosted web application that enables authenticated users to upload, organize, browse, preview, search, and manage images and files from any modern browser.

The application must run entirely on a Windows 11 PC using only free technologies while remaining portable enough to migrate to Linux or cloud infrastructure later without major code changes.

The experience should combine the simplicity of Google Photos with the organization of a lightweight file manager.

# 6. Non-Functional Requirements

## Performance

* Gallery first load <2 seconds on LAN.
* Thumbnail generation <1 second.
* API response <200 ms (excluding file transfer).
* Support at least 50 concurrent users.

## Security

* HTTPS via Cloudflare Tunnel
* JWT authentication
* Password hashing
* MIME validation
* Extension validation
* Path traversal protection
* Randomized stored filenames
* SHA-256 duplicate detection
* Rate limiting
* CORS protection
* Security headers

## Reliability

* Automatic startup on Windows boot.
* Graceful shutdown.
* Daily database backups.
* Structured logging.
* Health endpoint.

## Maintainability

* Modular architecture.
* Service layer.
* Repository pattern.
* Environment-based configuration.
* Storage abstraction layer.

# 10. File Processing Pipeline

1. Receive upload request.
2. Validate authentication.
3. Validate MIME type and extension.
4. Calculate SHA-256 hash.
5. Detect duplicates.
6. Store original file.
7. Generate thumbnail (background if needed).
8. Save metadata to SQLite.
9. Broadcast WebSocket event.
10. Display in gallery.

# 13. Deployment

The application runs directly on Windows 11.

Deployment steps:

1. Clone the repository.
2. Create a Python virtual environment.
3. Install backend dependencies.
4. Install frontend dependencies.
5. Configure `.env`.
6. Build the frontend.
7. Start FastAPI.
8. Configure Cloudflare Tunnel.
9. Register the backend as a Windows Service (recommended).

No containerization is required for Version 1.

---

# 14. Success Criteria

The application is considered production-ready when:

* Multiple authenticated users can upload files simultaneously.
* Images appear in the gallery automatically after upload.
* Images, PDFs, videos, and supported media open inside the web application.
* Search returns accurate results.
* Uploads are validated and stored safely.
* Daily backups complete successfully.
* The application can be accessed securely through Cloudflare Tunnel.
* The storage layer can later be replaced with cloud object storage without changing the frontend or public API.
